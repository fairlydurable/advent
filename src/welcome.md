# Welcome to Advent of Mojo 🔥

You learn a programming language by using it.

- Not by reading a language specification.  
- Not by memorizing every feature.  
- Not by studying type theory before writing code.

You learn a language when you start solving problems with it.

That is the goal of this book.

## Take a tour

Advent of Mojo is a hands-on tour of Mojo built around small, practical
programs. Each chapter takes on a task you'd hit during the first week
of Advent of Code:

- Work with files
- Work with strings
- Parse text into typed data
- Use lists, dictionaries, and sets
- Roll random numbers
- Sort and count values
- Iterate over data
- Break problems into functions
- Build simple reusable code

If you're looking at this page, we're guessing you already know how
to program.

You may come from Python, Swift, Rust, C++, JavaScript, Go, Java, or
another language entirely. You already understand variables, loops,
functions, and data structures. You don't need programming explained to
you again.

What you need is a working Mojo vocabulary.

This workbook helps you build that vocabulary through short explanations,
runnable examples, experiments, and exercises. The goal isn't deep Mojo
mastery and profound understanding of every Mojo feature.

This workbook's goal is fluency. By the time you reach the final pages,
you should feel comfortable reading and writing small Mojo programs
that actually do things.

You'll see some sharp edges along the way. That's because real programmers
deal with real code. This workbook won't hide things from you.

## About this workbook

Mojo is still evolving. Some features may feel unfamiliar. Some behaviors
may surprise you. When something matters, this book calls it out directly
instead of talking faster, waving hands, and giving you simplified
explanations.

Most chapters are designed to be read with a terminal open and the code
running beside you. Mojo is a systems language and it runs at the command
line. (You can use any editor you like.)

Read the examples. Change them. Break them. Try the experiments. Coding
builds muscle memory faster than passive reading.

This is a workbook, not a reference manual.

You won't need to memorize everything before moving forward. You can return
to chapters later as your understanding grows. We're putting syntax and
libraries first. Grow your working vocabulary. The concepts can wait.

For now, the goal is simple:

- Write code.  
- Solve problems.  
- Learn Mojo hands-on.

## Hands-on Mojo
